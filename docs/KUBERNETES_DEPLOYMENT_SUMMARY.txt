ARCHITECTURE
============

Kubernetes Deployment:
  6 StatefulSet pods → 6 different k8s nodes (anti-affinity)
    ↓
  Each pod: 4 virtual disks = 24 total disks
    ↓
  2 erasure sets (3 nodes each)
    ↓
  8+4 erasure coding (survives 4 disk failures)

Services:
  buckets-headless (ClusterIP: None)
    - DNS for inter-pod RPC: buckets-0.buckets-headless.buckets.svc
    
  buckets-lb (LoadBalancer)
    - External S3 API access
    - Session affinity for client experience

Storage:
  PersistentVolumeClaim per pod
    - 100Gi default (configurable)
    - ReadWriteOnce
    - Survives pod restarts

CONFIGURATION
=============

Adjust Resources (statefulset.yaml):
  resources:
    requests:
      cpu: 500m
      memory: 1Gi
    limits:
      cpu: 2000m
      memory: 4Gi

Adjust Storage (statefulset.yaml):
  volumeClaimTemplates:
    - spec:
        resources:
          requests:
            storage: 100Gi  # Change this
        storageClassName: fast  # Change this

Adjust Replicas (statefulset.yaml):
  spec:
    replicas: 6  # Change this
  
  Also update configmap.yaml with new nodes

TROUBLESHOOTING
===============

Pods Not Starting:
  - Check: kubectl describe pod buckets-0 -n buckets
  - Fix: Verify sufficient nodes (need 6 for anti-affinity)
  - Fix: Check storage class exists

PVCs Pending:
  - Check: kubectl get pvc -n buckets
  - Fix: Update storageClassName in statefulset.yaml
  - Fix: Ensure storage provisioner is available

No LoadBalancer IP:
  - Check: kubectl get svc buckets-lb -n buckets
  - Fix: Use NodePort or port-forward if cloud doesn't support LB
  - Fix: kubectl patch svc buckets-lb -n buckets -p '{"spec":{"type":"NodePort"}}'

Format Errors:
  - Fix: kubectl exec -n buckets buckets-0 -- /app/bin/buckets format --config /etc/buckets/config.json
